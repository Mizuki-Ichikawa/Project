--ルール--
・各自それぞれのブランチ（作業スペースのようなもの）を作成しています。
・mainのファイルの変更は行わないでください。
・各自のブランチの変更のみを行ってください。
・各自のブランチに関しては、変更、コミット、pushは自由にしてくれてOKです。
・間違えて、main、他の人のブランチを変更してしまった場合は速やかに連絡してください。

--各コマンドの説明--
1, 各自のブランチに移動するコマンド（超重要）
    git checkout name  (nameのところは自分のブランチ名　ex: ミズキのブランチ->Mizuki)
    まず最初にこれを絶対に行うこと！！！　これを行わないとmainの変更を加えることになるから注意！！
2, 変更点を自分のvscode内に保存する
    手順1：vscodeの左側にある３つの〇が線でつながれてるところをクリックする
    手順2：メッセージ欄にどんな変更をしたかの題名を記入する。
    手順3：✓のボタンもしくは✓コミットと書かれた青色の領域をクリックする
    
    これを行うと自分のvscodeのソース管理グラフのところに変更部分の履歴が残される。
    作業中にエラーが起きたときは、履歴をさかのぼると原因が見つけやすくなる。
    この時点では、自分のvscode内で変更がされただけでGitHub上では変更されていないため注意！
3, 変更点をGitHub上に反映させる
    手順1：ソース管理の・・・（その他の操作）からプッシュをクリックする。
    手順2：GitHubにログインして、変更が反映されているかを確認する。

    こうすることで、自分が変更した内容をほかの人が見ることができるようになる！！

4, mainでの変更をvscode上に保存する
    これは、mainで変更があった際にのみ使用する。
    手順1：git checkout main　をターミナルに入力して、mainブランチに移動する。
    手順2：git pull　をターミナルで入力する　

    これで、mainでの変更が自分のvscode上で保存されることになる。
    ※注意※
    mainで変更があった際にpullせずに誤って自分が変更してしまうと競合が発生してしまうので気を付けて！
    これを避けるために、mainの変更を禁止しています。
